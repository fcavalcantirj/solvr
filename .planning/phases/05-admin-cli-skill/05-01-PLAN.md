# Plan 05-01: Admin CLI Skill (solvr-admin.sh)

## Objective

Create a Claude Code-callable bash script `solvr-admin.sh` that wraps the admin HTTP API for email broadcast operations. The script follows the exact patterns established in `solvr.sh`: `load_api_key()`, `api_call()`, subcommand dispatch via `case`, `jq` for JSON, and color output. Key differences: uses `ADMIN_API_KEY` / `X-Admin-API-Key` header, connects to `/admin/` routes (no `/v1/` prefix), and reads from `~/.config/solvr/admin-credentials.json`.

## must_haves

- `load_admin_key()` reads `ADMIN_API_KEY` env var first, then `~/.config/solvr/admin-credentials.json` (field `admin_api_key`); exits non-zero with clear error if neither found — TOOL-04
- `api_call()` uses `-H "X-Admin-API-Key: ${admin_key}"` (not `Authorization: Bearer`) and base URL `https://api.solvr.dev`
- `solvr-admin email send --subject "X" --body "Y"` → `POST /admin/email/broadcast` with `{"subject":"X","body_text":"Y","body_html":"<p>Y</p>"}` — TOOL-01
- `solvr-admin email dry-run --subject "X" --body "Y"` → same endpoint with `{"dry_run":true,...}` — TOOL-02
- `solvr-admin email history` → `GET /admin/email/history`, formatted table output — TOOL-03
- `--body-html` flag on `send` and `dry-run` allows passing raw HTML (overrides auto-wrap); `--body` sets `body_text` and auto-wraps for `body_html`
- `--json` flag on all subcommands prints raw JSON response
- Key NEVER appears in shell history; always passed via HTTP header
- Script exits non-zero on HTTP 4xx/5xx errors with human-readable message from response body
- `chmod 600` enforced on credentials file when written (not written by this script, but documented)

# solvr-admin.sh - Admin CLI for Solvr email broadcast operations

set -euo pipefail

SOLVR_ADMIN_API_URL="${SOLVR_ADMIN_API_URL:-https://api.solvr.dev}"

SOLVR_ADMIN_CREDENTIALS_FILE="${SOLVR_CONFIG_DIR}/admin-credentials.json"

**2. Color block** — identical to `solvr.sh` (terminal detection, RED/GREEN/YELLOW/BLUE/CYAN/NC).

**3. `load_admin_key()` function:**

```bash
load_admin_key() {
    local admin_key=""

    # Priority 1: Environment variable
    if [ -n "${ADMIN_API_KEY:-}" ]; then
        admin_key="$ADMIN_API_KEY"
    # Priority 2: Admin credentials file
    elif [ -f "$SOLVR_ADMIN_CREDENTIALS_FILE" ]; then
        admin_key=$(jq -r '.admin_api_key // empty' "$SOLVR_ADMIN_CREDENTIALS_FILE" 2>/dev/null || echo "")
    fi

    if [ -z "$admin_key" ]; then
        echo -e "${RED}Error: No admin API key found${NC}" >&2
        echo "Set ADMIN_API_KEY environment variable or create ${SOLVR_ADMIN_CREDENTIALS_FILE}" >&2
        echo "File format: {\"admin_api_key\": \"your-key-here\"}" >&2
        echo "Permissions: chmod 600 ${SOLVR_ADMIN_CREDENTIALS_FILE}" >&2
        return 1
    fi

    echo "$admin_key"
}
```

**4. `api_call()` function** — same structure as `solvr.sh` but with admin auth header:

```bash
api_call() {
    local method="$1"
    local endpoint="$2"
    local data="${3:-}"
    local admin_key

    admin_key=$(load_admin_key) || return 1

    local url="${SOLVR_ADMIN_API_URL}${endpoint}"
    local curl_args=(
        -s
        -X "$method"
        -H "X-Admin-API-Key: ${admin_key}"
        -H "Content-Type: application/json"
        -H "Accept: application/json"
        -w "\n%{http_code}"
    )

    if [ -n "$data" ]; then
        curl_args+=(-d "$data")
    fi

    local response
    response=$(curl "${curl_args[@]}" "$url")

    local http_code
    http_code=$(echo "$response" | tail -n1)
    local body
    body=$(echo "$response" | sed '$d')

    if [ "$http_code" -ge 400 ]; then
        local error_msg
        error_msg=$(echo "$body" | jq -r '.error // .message // "Unknown error"' 2>/dev/null || echo "Request failed")
        echo -e "${RED}Error ($http_code): ${error_msg}${NC}" >&2
        return 1
    fi

    echo "$body"
}
```

**5. `cmd_email()` function** — top-level dispatcher for the `email` subcommand:

```bash
cmd_email() {
    local subcmd="${1:-}"
    shift || true

    case "$subcmd" in
        send)    cmd_email_send "$@" ;;
        dry-run) cmd_email_dryrun "$@" ;;
        history) cmd_email_history "$@" ;;
        *)
            echo "Usage: solvr-admin email <send|dry-run|history>"
            echo ""
            echo "Subcommands:"
            echo "  send    --subject <s> --body <b> [--body-html <h>] [--json]"
            echo "  dry-run --subject <s> --body <b> [--body-html <h>] [--json]"
            echo "  history [--json]"
            ;;
    esac
}
```

**6. `cmd_email_send()` function** — handles `email send`:

Parse `--subject`, `--body`, `--body-html`, `--json` flags using a `while` loop with `case` (same pattern as `solvr.sh cmd_search`). Validate that `--subject` and `--body` (or `--body-html`) are provided; exit 1 with error if missing.

Build payload via `jq -n`:
- `body_text` = value of `--body` (empty string if not given)
- `body_html` = value of `--body-html` if given; otherwise `"<p>${body_text}</p>"` (use `jq --arg` to inject)
- `dry_run` = `false`

Call `api_call POST "/admin/email/broadcast" "$payload"`.

On success, if `--json`: print raw JSON. Otherwise print:
```
Broadcast complete.
  Broadcast ID: <broadcast_id from .broadcast_id>
  Sent:         <.sent>
  Failed:       <.failed>
  Total:        <.total>
  Duration:     <.duration_ms>ms
```
Use `${GREEN}` for "Broadcast complete." line.

**7. `cmd_email_dryrun()` function** — handles `email dry-run`:

Identical flag parsing to `cmd_email_send`. Build payload the same way but set `dry_run` = `true`.

Call `api_call POST "/admin/email/broadcast" "$payload"`.

On success, if `--json`: print raw JSON. Otherwise print:
```
Dry-run preview (nothing sent).
  Would send to: <.would_send> recipients

Recipients:
```
Then iterate `.recipients[]` with `jq -r` to print each email address, one per line with `  - ` prefix.
Use `${YELLOW}` for the header line to distinguish from a real send.

**8. `cmd_email_history()` function** — handles `email history`:

Parse `--json` flag only.

Call `api_call GET "/admin/email/history"`.

On success, if `--json`: print raw JSON. Otherwise print a formatted table using `jq -r`:

```
Past Broadcasts:
  ID                                   Subject            Sent  Failed  Status     Started At
  ─────────────────────────────────────────────────────────────────────────────────────────────
```
Then for each item in the response array, print:
```
  <broadcast_id | head 8 chars>...  <subject | 18 chars padded>  <sent_count>  <failed_count>  <status>  <started_at | date only>
```

Use `jq -r '.[] | [.broadcast_id[0:8], .subject[0:18], (.sent_count|tostring), (.failed_count|tostring), .status, (.started_at | split("T")[0])] | @tsv'` and format with `column -t` if available, otherwise raw tsv.

If the array is empty, print `  No broadcasts found.` in YELLOW.

**9. `cmd_help()` function:**

```
solvr-admin - Admin CLI for Solvr email broadcast

USAGE:
    solvr-admin <command> [subcommand] [options]

COMMANDS:
    email send    --subject <s> --body <b> [--body-html <h>] [--json]
                  Broadcast email to all active users
    email dry-run --subject <s> --body <b> [--body-html <h>] [--json]
                  Preview recipients without sending
    email history [--json]
                  List past broadcasts
    help          Show this help

FLAGS:
    --subject <s>    Email subject (required for send/dry-run)
    --body <b>       Plain text body; auto-wrapped as <p>text</p> for HTML
    --body-html <h>  Raw HTML body (overrides --body for HTML part)
    --json           Output raw JSON response

AUTHENTICATION (in priority order):
    1. ADMIN_API_KEY environment variable
    2. ~/.config/solvr/admin-credentials.json  (field: admin_api_key)

SECURITY:
    The admin key is passed via X-Admin-API-Key header, never as a URL param.
    Credentials file must be chmod 600.

EXAMPLES:
    # Send a broadcast
    solvr-admin email send --subject "Platform update" --body "Hello everyone..."

    # Preview recipients before sending
    solvr-admin email dry-run --subject "Test" --body "Hello"

    # Send with rich HTML
    solvr-admin email send --subject "Update" --body "See below." \
        --body-html "<h1>Platform Update</h1><p>See below.</p>"

    # View broadcast history
    solvr-admin email history

    # Get raw JSON for scripting
    solvr-admin email history --json | jq '.[0]'
```

**10. `main()` function** — top-level dispatch:

```bash
main() {
    if [ $# -eq 0 ]; then
        cmd_help
        exit 0
    fi

    local command="$1"
    shift

    case "$command" in
        email)
            cmd_email "$@"
            ;;
        help|--help|-h)
            cmd_help
            ;;
        *)
            echo -e "${RED}Error: Unknown command: ${command}${NC}" >&2
            echo "Run 'solvr-admin help' for usage information" >&2
            exit 1
            ;;
    esac
}

main "$@"
```
