# Plan 02-02: Resend Client + EmailSender Interface

## Objective

Build a `ResendClient` struct that wraps the `resend-go/v3` SDK and satisfies a minimal `EmailSender` interface. The client sends emails via the Resend API with the `FROM_EMAIL` address baked in at construction time. Unit tests use `httptest.NewServer` to verify the HTTP request shape without hitting the real Resend API.

## must_haves

- `ResendClient` struct exists in `backend/internal/services/resend.go`
- `ResendClient` has a `Send(ctx, to, subject, htmlBody, textBody string) error` method
- `Send` uses `FROM_EMAIL` value passed at construction (default `noreply@solvr.dev`)
- `Send` calls Resend API with correct `Authorization: Bearer <key>` header
- `Send` sets `From` field to `Solvr <noreply@solvr.dev>` (or configured FROM_EMAIL)
- Unit test passes without network access (mock HTTP server)
- `resend-go/v3` is added to `go.mod` via `go get`
- `go build ./cmd/api` compiles successfully after adding the import

**IMPORTANT:** Do NOT use the resend-go SDK directly in the `Send` method if the SDK does not support overriding the base URL for testing. Instead, use direct HTTP calls to the Resend API (`POST /emails`). This gives full control over the base URL for unit tests.

If the SDK does support base URL override (check the SDK source: look for `BaseURL` field on the client struct), then use the SDK.

**Direct HTTP approach (preferred for testability):**

```go
package services

import (
	"bytes"
	"context"
	"encoding/json"
	"fmt"
	"io"
	"net/http"
)

const defaultResendBaseURL = "https://api.resend.com"

// ResendClient sends emails via the Resend API.
// It implements the EmailSender interface (defined in handlers package)
// via Go's implicit interface satisfaction.
type ResendClient struct {
	apiKey    string
	fromEmail string
	baseURL   string
	client    *http.Client
}

// NewResendClient creates a new ResendClient.
// apiKey is the Resend API key (RESEND_API_KEY env var).
// fromEmail is the sender address (e.g., "noreply@solvr.dev").
func NewResendClient(apiKey, fromEmail string) *ResendClient {
	return &ResendClient{
		apiKey:    apiKey,
		fromEmail: fromEmail,
		baseURL:   defaultResendBaseURL,
		client:    &http.Client{},
	}
}

// SetBaseURL overrides the Resend API base URL (for testing with httptest).
func (c *ResendClient) SetBaseURL(url string) {
	c.baseURL = url
}
```

The `Send` method:

```go
// Send sends a single email via the Resend API.
// Parameters: to (recipient), subject, htmlBody, textBody (can be empty).
// The from address is set at construction time.
func (c *ResendClient) Send(ctx context.Context, to, subject, htmlBody, textBody string) error {
```

Implementation details:
- Build JSON payload with fields: `from` (format: `"Solvr <fromEmail>"`), `to` (as `[]string{to}`), `subject`, `html`, `text` (omit if empty)
- POST to `c.baseURL + "/emails"`
- Set headers: `Authorization: Bearer <apiKey>`, `Content-Type: application/json`
- Use `http.NewRequestWithContext(ctx, ...)` for context support
- If response status is not 2xx, read body and return `fmt.Errorf("resend API error (status %d): %s", resp.StatusCode, body)`
- On success, return nil (we don't need the email ID for now)

The `from` field format should be `"Solvr <noreply@solvr.dev>"` — wrapping the email in a display name. This is standard email formatting.
