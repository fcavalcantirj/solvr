# Plan 12-02: Content Endpoints — Remove Generic CRUD, Fix Accuracy, Add Missing Content Endpoints

<objective>
Rewrite api-endpoint-data-content.ts to remove generic POST/PATCH/DELETE /posts (use type-specific instead), fix all accuracy issues (evolve params/response, responses missing param, vote response, DELETE 204s), and add missing content endpoints (GET /posts/{id}/my-vote, approach history). Keep GET /posts/{id} for reading any post by ID. Result: agent-first content docs matching actual backend.
</objective>

<must_haves>
- Generic POST /posts is GONE
- Generic PATCH /posts/{id} is GONE
- Generic DELETE /posts/{id} is GONE
- Generic GET /posts list is GONE
- GET /posts/{id} (read single post) REMAINS
- POST /ideas/{id}/evolve has correct params: evolved_post_id (not title/description)
- POST /ideas/{id}/evolve response is { message, idea_id, evolved_post_id } (not { id, evolved_from })
- POST /ideas/{id}/responses has response_type param (required)
- POST /posts/{id}/vote response includes user_vote field
- All example IDs use UUIDs, not p_abc123 format
- POST /approaches/{id}/verify endpoint exists with correct params
- POST /approaches/{id}/progress endpoint exists with correct params
- GET /problems/{id}/export endpoint exists
- GET /posts/{id}/my-vote endpoint exists
- GET /problems/{id}/approaches/{aid}/history endpoint exists
- File stays under ~900 lines
</must_haves>
