## Typography

Solvr uses a two-family system: Inter (sans) for prose, JetBrains Mono for UI chrome.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Eyebrow | 10px (`text-[10px]`) | 400 (regular) | 1 | Mono | Section labels above headings ŌĆö `tracking-[0.3em]`, all-caps (e.g., "LIVE SEARCH ACTIVITY") |
| Body / description | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-relaxed`) | Sans | Muted description text in page headers, card descriptions |
| Label / nav / CTA | 12px (`text-xs`) | 400 (regular) | 1 | Mono | Navigation links, table column headers, CTA buttons ŌĆö `tracking-wider`, all-caps |
| Page heading | 36ŌĆō48px (`text-4xl md:text-5xl`) | 300ŌĆō400 (light/normal) | 1.1 (`tracking-tight`) | Sans | Page h1 (e.g., "Live Search Activity") |

Rules:
- All navigation and CTA text: `font-mono text-xs tracking-wider` uppercase
- Stat card metric values (large numbers): `text-3xl font-light tracking-tight font-sans`
- Table cell content (trending queries): `text-sm font-sans` at weight 400
- Percentage/secondary values in stat cards: `text-sm font-mono text-muted-foreground`
- Chart axis labels: `text-[10px] font-mono text-muted-foreground` (matches recharts default tick style)

## Interaction Contracts

### Live Indicator

- Green dot: `w-2 h-2 rounded-full bg-green-500 animate-pulse`
- Placed inline with page eyebrow text: `[ŌŚÅ Live]` using `inline-flex items-center gap-2`
- On data refresh: entire data section fades via `transition-opacity duration-500`
- Stat card numbers animate on value change: use CSS `transition-all duration-700` on the number element

### Time Range Toggle (D-09)

- Use shadcn `Tabs` component with `TabsList` + three `TabsTrigger` items: "1h", "24h", "7d"
- Default selected: "24h"
- On tab change: trigger immediate re-fetch, show Skeleton for 300ms minimum (avoid flash)
- Unselected tabs: `text-muted-foreground`, selected: `bg-background text-foreground` (Tabs default)

### Bot Filter Toggle (D-16)

- Small toggle below stat cards: `font-mono text-xs text-muted-foreground` label "Show automated searches"
- Use shadcn `Switch` component, default off (filtered view is default)
- On toggle: re-fetch all three endpoints with `?include_bots=true` or omit param

### Loading Skeleton

Show `Skeleton` component during initial load and on window change:
- Stat cards: 4x skeleton cards, `h-24` each, same grid as real cards
- Trending table: 10x skeleton rows, `h-8` each
- Charts: 2x skeleton blocks, `h-48` each
- Minimum skeleton display: 300ms (prevents flash on fast connections)

### Empty/Sparse State (Pitfall 6)

When API returns empty array for selected window (e.g., 1h):
- Replace table and charts with `Empty` component
- Title: "No activity in this window"
- Description: "No searches recorded in the last 1h. Try the 24h view."
- Do NOT render empty recharts axes (shows broken chart)

### Post Type Selector (D-17 + D-21)

In `new-post-form.tsx`: change `grid-cols-3` to `grid-cols-2` when question type is removed.
No visual treatment needed ŌĆö simply omit the question card from the array.
