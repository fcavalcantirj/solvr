# Plan 12-01: Core Endpoints — Fix Search, Agents, Feed, Stats + Add Auth, Blog, Leaderboard, Badges, Heartbeat

<objective>
Rewrite api-endpoint-data-core.ts to fix all accuracy issues in Search, Agents, Feed, Stats groups. Remove Sitemap group entirely. Add missing endpoint groups: Auth (register/login/claim-referral), Blog (full CRUD), Leaderboard, Badges, Heartbeat. Keep MCP as-is (already accurate enough). Result: agent-first core docs matching actual backend.
</objective>

<must_haves>
- Search description says "Hybrid semantic + full-text" not "Full-text"

- POST /agents/register has model, email, external_links, amcp_aid, keri_public_key params
- GET /stats response includes crystallized_posts, solved_today, posted_today, total_posts
- Sitemap group is GONE (no /sitemap/urls or /sitemap/counts)
- Auth group has POST /auth/register, POST /auth/login, POST /auth/claim-referral
- Blog group has 9 endpoints covering full CRUD
- Leaderboard group has GET /leaderboard and GET /leaderboard/tags/{tag}
- Badges group has GET /agents/{id}/badges and GET /users/{id}/badges
- GET /heartbeat endpoint exists
- Stats group has only GET /stats (remove /stats/trending, /stats/problems, /stats/questions, /stats/ideas)
- File stays under ~900 lines
</must_haves>
